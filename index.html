Enfoque del Proyecto

El presente proyecto tiene como objetivo analizar, diseñar y proponer una solución digital para una pizzería que actualmente gestiona pedidos de manera manual.

Se propone el desarrollo de una aplicación mobile-first, que permita digitalizar el proceso de pedidos, integrar pagos online y mejorar la experiencia del cliente.

Objetivos

Objetivo General:
Desarrollar un Sistema Online de Pedidos que permita optimizar la Gestión Operacional del Negocio.
Objetivos Específicos:
  • Reducción de Errores
  • Implementar el Sistema de Pagos Digitales
  • Mejora en Tiempos de Gestión Operativa
  • Automatizaciones de Procesos

Restricciones de un Proyecto

Tradicionalmente, la gestión de proyectos se basa en tres variables fundamentales
  • Alcance: Define las funcionalidades y entregables del sistema
  • Tiempo: Establece la duración del proyecto
  • Costo: Representa los recursos económicos necesarios
Estas tres variables están interrelacionadas, por lo que cualquier modificación en una de ellas impacta en las otras.
Observación Práctica: Si se reduce el tiempo de desarrollo, será necesario aumentar los recursos (costo) o disminuir el alcance del sistema.

Nueva Visión de Gestión de Proyectos

Los modelos modernos de gestión de proyectos amplían la visión tradicional incorporando nuevas dimensiones:
  • Calidad
  • Recursos
  • Riesgos
  • Satisfacción del cliente
Esto permite una evaluación más completa del éxito del proyecto.

Implicancias del Proyecto

En el presente proyecto, las restricciones se aplican de la siguiente manera
Se utilizan indicadores clave de desempeño (KPI) para evaluar el progreso
  • Calidad: El sistema debe funcionar correctamente y sin errores críticos
  • Recursos: Se consideran herramientas tecnológicas y tiempo del equipo de desarrollo
  • Riesgos: Integración de pagos, tiempos de entrega, adopción del sistema
  • Satisfacción del cliente: Facilidad de uso, rapidez y confiabilidad (Comodidad del Usuario)

Se aplica gestión de cambios para controlar modificaciones en el alcance y evitar desviaciones en tiempo y costo.

Conclusión El éxito del proyecto no depende únicamente de cumplir con el tiempo y el presupuesto, sino de entregar una solución que aporte valor real al negocio y satisfaga las necesidades del usuario final. Un sistema que no cumpla con estas expectativas, aunque respete los plazos y costos, no puede considerarse exitoso desde una perspectiva estratégica.
